## О проекте

Этот репозиторий GitHub содержит торговую систему, предназначенную для автоматизированной алгоритмической торговли на Binance Futures, Bybit, BitMEX и FTX.

Данная торговая система предлагает удобную платформу для тестирования торговых стратегий с помощью бэктестинга и бумажной торговли (paper trading), а также для выполнения реальных сделок. Система минимизирует расхождения между симулированной и реальной торговлей, обеспечивая бесшовный переход от бэктестинга к бумажной торговле и затем к реальной торговле. При переходе от симулированной к реальной торговле изменения в коде стратегии минимальны.

При разработке стратегий от пользователей ожидается базовое понимание трейдинга; при этом на них не накладывается множество ограничений. Система построена на предопределенных событиях, таких как обновления рыночных данных, обновления ордеров и исполнение сделок, и предоставляет различные технические возможности, включая продвинутые типы ордеров, а также мониторинг позиций и ордеров в реальном времени.

## Возможности

- Реализация REST API и WebSocket для нескольких криптовалютных бирж, включая Bybit, BitMEX, Binance Futures и FTX.
- Предопределенные события для обновлений рыночных данных, обновлений ордеров и исполнения сделок.
- Все типы ордеров с поддержкой большинства параметров/комбинаций.
- Поддержка сложных типов ордеров: лимитные ордера, стоп-лосс ордера, трейлинг-стоп ордера, айсберг-ордера и ордера с "догоняющим" лимитом (limit chasing orders).
- Реализация пользовательских стратегий, позволяющая трейдерам определять собственные торговые алгоритмы и индикаторы.
- Фреймворк для бэктестинга стратегий на исторических рыночных данных.
- Тестовые сети (testnet) для Bybit, BitMEX и Binance Futures, позволяющие тестировать стратегии без риска для реальных средств.
- Бумажная торговля (paper trading / stub trading) для симуляции сделок с фальшивыми средствами.
- Индикаторы TA-lib и многие другие, включая пользовательские.
- Простая реализация стратегий, позволяющая переносить стратегии с других платформ, таких как TradingView.
- Поддержка вебхуков Discord и уведомлений LINE для получения обновлений в реальном времени.

## Как установить

### 1. Установка пакетов

#### OSX

```bash
$ brew install ta-lib
$ git clone https://github.com/TheFourGreatErrors/alpha-rptr.git
$ cd alpha-rptr/
$ pip install -r requirements.txt
```

#### LINUX

```bash
$ wget http://prdownloads.sourceforge.net/ta-lib/ta-lib-0.4.0-src.tar.gz
$ tar xvzf ta-lib-0.4.0-src.tar.gz
$ cd ta-lib/
$ ./configure --prefix=/usr
$ make
$ sudo make install
$ git clone https://github.com/TheFourGreatErrors/alpha-rptr.git
$ cd alpha-rptr/
$ pip install -r requirements.txt
```

**Примечание:** Мы используем Python-обертку для библиотеки `TA-Lib`, которая предоставляет широкий спектр функций технического анализа для финансовых рынков. Базовая библиотека TA-Lib написана на языке C. Поэтому для установки и использования этой библиотеки ее необходимо правильно скомпилировать в вашей системе. Python-обертка позволяет использовать эти функции из кода Python, но процесс установки может потребовать некоторых технических знаний. Важно убедиться, что C-библиотека правильно скомпилирована перед установкой, чтобы избежать ошибок и обеспечить корректную работу библиотеки.

### 2. Настройка ключей

Файл `src/config.py` предназначен для установки ваших API-ключей и других настроек конфигурации для торгового бота. Вот как это сделать:

1. Откройте файл config.py в текстовом редакторе или IDE.
2. Найдите раздел `binance_keys` в словаре конфигурации. Сюда вы введете свои API-ключи Binance.
3. Замените пустую строку `""` в полях `API_KEY` и `SECRET_KEY` на ваши реальные API-ключ и секретный ключ Binance соответственно. Вы можете найти свои API-ключи, войдя в свою учетную запись Binance и перейдя на страницу управления API.
4. Если у вас есть дополнительные учетные записи Binance или других бирж, вы можете ввести их API-ключи в соответствующих разделах словаря конфигурации.
5. Сохраните файл config.py.

Помимо API-ключей, вы можете установить другие настройки конфигурации, такие как URL-адреса вебхуков для Discord и LINE, а также параметры проверки работоспособности (health checks) для мониторинга состояния ваших аккаунтов.

Обратите внимание, что вы также можете настроить различные торговые профили, используя поле `args_profile`, которое позволяет указать разные настройки для разных торговых сессий. Чтобы использовать определенный профиль, вы можете запустить программу с флагом `--profile <имя профиля>`.

## Как запустить

Чтобы использовать торгового бота, вам нужно запустить скрипт main.py с соответствующими параметрами. Вот разбор доступных опций:

- `--account`: Указывает аккаунт для торговли. Должен соответствовать имени файла конфигурации в директории config. Например, если у вас есть файл `binanceaccount1` в директории config, вы должны использовать `--account binanceaccount1`.
- `--exchange`: Указывает биржу для использования. В настоящее время бот поддерживает Binance Futures, BitMEX, Bybit и FTX. Используйте `--exchange binance` для торговли на Binance Futures или `--exchange bitmex` для торговли на BitMEX.
- `--pair`: Указывает торговую пару. Это должна быть допустимая торговая пара на выбранной бирже. Например, `--pair BTCUSDT` укажет пару Bitcoin/USDT на Binance Futures.
- `--strategy`: Указывает торговую стратегию. Должно соответствовать имени Python-файла в директории strategies. Например, если у вас есть файл `sample_strategy.py` в директории strategies (чувствительно к регистру), вы должны использовать `--strategy sample_strategy`.

Чтобы выполнить торгового бота с указанными параметрами, запустите следующую команду:

```bash
$ python main.py --account binanceaccount1 --exchange binance --pair BTCUSDT --strategy Sample
```

Например, если вы хотите использовать стратегию Channel Breakout на BitMEX с bitmexaccount1 и парой XBTUSD, вы должны выполнить следующую команду:

```bash
$ python main.py --account bitmexaccount1 --exchange bitmex --pair XBTUSD --strategy Doten
```

Бот также поддерживает другие режимы работы, такие как бэктестинг, демо-торговля на тестовой сети, гипероптимизация параметров и бумажная торговля (stub trading). Для получения дополнительной информации об этих режимах и их соответствующих опциях выполните следующую команду:

```bash
$ python main.py --help
```

## Режимы работы

### 1. Режим реальной торговли (Production)

В этом режиме скрипт будет выполнять реальные сделки на бирже Binance для указанной учетной записи и торговой пары с использованием заданной стратегии. Чтобы запустить скрипт в этом режиме, используйте следующую команду:

```bash
$ python main.py --account binanceaccount1 --exchange binance --pair BTCUSDT --strategy Sample
```

### 2. Демо-режим

В этом режиме скрипт будет выполнять демо-сделки на [BitMEX testnet](https://testnet.bitmex.com/), [Binance Futures testnet](https://testnet.binancefuture.com/en/futures/BTCUSDT) или [Bybit testnet](https://testnet.bybit.com/en-US/) для указанной учетной записи и торговой пары с использованием заданной стратегии. Чтобы запустить скрипт в этом режиме, используйте следующую команду:

```bash
$ python main.py --demo --account bitmexaccount1 --exchange bitmex --pair XBTUSD --strategy Sample
```

### 3. Режим бэктестинга

В этом режиме скрипт выполнит бэктестинг указанной стратегии с использованием исторических данных для заданной торговой пары на бирже Binance. Чтобы запустить скрипт в этом режиме, используйте следующую команду:

```bash
$ python main.py --test --account binanceaccount1 --exchange binance --pair BTCUSDT --strategy Sample
```

### 4. Режим гипероптимизации (Hyperopt)

Режим Hyperopt автоматически ищет наилучшие значения гиперпараметров для оптимизации производительности торговой стратегии. Чтобы запустить скрипт в этом режиме, используйте следующую команду:

```bash
$ python main.py --hyperopt --account binanceaccount1 --exchange binance --pair BTCUSDT --strategy Sample
```

### 5. Режим бумажной торговли (Stub trade mode)

В этом режиме скрипт будет симулировать сделки на бирже Binance для указанной учетной записи и торговой пары с использованием заданной стратегии. Реальные сделки выполняться не будут. Чтобы запустить скрипт в этом режиме, используйте следующую команду:

```bash
$ python main.py --stub --account binanceaccount1 --exchange binance --pair BTCUSDT --strategy Sample
```

## Как добавить собственную стратегию

Чтобы добавить новую стратегию в торгового бота, выполните следующие шаги:

- Создайте новый файл в папке `src/strategies` с именем, точно совпадающим с именем вашего класса стратегии (чувствительно к регистру).
- Импортируйте необходимые файлы, такие как индикаторы, из других примеров стратегий или импортируйте свои собственные.
- Определите класс вашей стратегии в новом файле.
- Реализуйте логику вашей стратегии внутри метода `strategy` вашего класса.
- Используйте объект `self.exchange` для выполнения ордеров на основе сигналов вашей стратегии.
- При необходимости определите другие методы, которые могут потребоваться вашей стратегии.

Этот пример поможет вам начать работу с добавлением собственных стратегий в бота.

```python
# пример стратегии
class Sample(Bot):
    def __init__(self): 
        # установите таймфрейм здесь
        Bot.__init__(self, ['15m'])
        # инициализируйте переменные
        self.isLongEntry = []
        self.isShortEntry = []
        
    # Настройка гипероптимизации
    # Два гиперпараметра, которые мы оптимизируем: 'fast_len' и 'slow_len'.
    # 'fast_len' представляет параметр длины для быстрой операции, а 'slow_len' - параметр длины для медленной операции.
    # Оба гиперпараметра имеют равномерное распределение между 1 и 20 для 'fast_len' и между 1 и 30 для 'slow_len' соответственно, с шагом 1.
    # Hyperopt будет исследовать различные комбинации этих гиперпараметров, чтобы найти наилучшую конфигурацию, максимизирующую производительность нашей модели.
    def options(self):
        return {
            'fast_len': hp.quniform('fast_len', 1, 20, 1),
            'slow_len': hp.quniform('slow_len', 1, 30, 1),
        }
    
    # Переопределите эту функцию класса bot, чтобы установить длину исторических данных свечей, необходимых для ваших индикаторов
    # В нашем случае самый длинный индикатор требует 18 исторических значений свечей, поэтому 100 более чем достаточно
    def ohlcv_len(self):
        return 100

    def strategy(self, action, open, close, high, low, volume):    
        # это ваша функция стратегии
        # используйте аргумент action для реализации мультитаймфрейма, так как строка таймфрейма будет передана как `action`
        
        # Определите размер лота для ваших ордеров
```

##### Наиболее распространенные типы ордеров с примерами

Вот краткие описания наиболее распространенных типов рыночных ордеров:

1. **Рыночный ордер (Market Order):** Ордер на покупку или продажу актива по наилучшей доступной цене на рынке. Исполняется немедленно по текущей рыночной цене.

2. **Лимитный ордер (Limit Order):** Ордер на покупку или продажу актива по определенной цене или лучше. Будет исполнен только по указанной цене или лучше.

3. **Стоп-лимитный ордер (Stop Limit Order):** Ордер на покупку или продажу актива при достижении ценой определенного уровня, но только если сделка может быть исполнена по определенной лимитной цене или лучше.

4. **Стоп-рыночный ордер (Stop Market Order):** Ордер на покупку или продажу актива при достижении ценой определенного уровня. Исполняется как рыночный ордер после достижения стоп-цены.

5. **Ордер только на уменьшение (Reduce-Only Order):** Этот ордер может только уменьшить размер вашей позиции или закрыть сделку. Вы не можете увеличить свою позицию с помощью reduce-only ордера.

6. **Ордер только на добавление (Post-Only Order):** Этот ордер только размещается в книге ордеров и не будет исполнен немедленно. Полезен для трейдеров, которые хотят получить комиссию мейкера вместо комиссии тейкера.

7. **Ордер с догоняющим лимитом (Limit Chasing Order):** Это лимитный ордер, который движется вместе с рыночной ценой. Полезен для трейдеров, которые хотят купить по наилучшей возможной цене, не обновляя постоянно свой лимитный ордер.

8. **Айсберг-ордер (Iceberg Order):** Это крупный ордер, который разбивается на более мелкие ордера и исполняется с течением времени. Меньшие ордера не видны рынку, поэтому трейдер может избежать влияния на рыночную цену крупным ордером.

**Важно отметить**, что разные торговые платформы могут иметь свои собственные типы ордеров и вариации.

Эти примеры обычно находятся в контексте `strategy()`, который находится внутри класса Bot, поэтому мы обращаемся к нему через `self.exchange`:

## Сохранение состояния сессии стратегии

Иногда нам может потребоваться перезапустить стратегии со сложным внутренним состоянием, и мы можем захотеть сохранить это состояние между перезапусками.

Для этого по умолчанию в каждой стратегии предусмотрена специальная возможность. Вы можете определить переменные стратегии, которые хотите сохранить, внутри функции `init()` стратегии как `self.session.<whatever> = <whatever>`, и эти переменные, находящиеся в пространстве имен session, будут опционально сохранены в JSON-файл при выходе из бота.

Вы можете включить сохранение сессии, добавив `--session <filename.json>` в команду оболочки для запуска бота. Затем бот запросит, загружать ли сессию, если она существует, или создаст новую. Вы можете проигнорировать сохраненную сессию, и бот сохранит сессию заново в конце.

## Продвинутое использование сессий

Внутри стратегии вы можете управлять сессией до сохранения и загрузки, переопределяя методы `get_session()` и `set_session(session)`.

Это полезно, поскольку сохранение с помощью JSON поддерживает только базовые типы, а у вас могут быть другие типы, которые необходимо сохранить. Переопределение set_session и get_session позволяет вручную обрабатывать преобразование из базовых типов и обратно.

## Логирование метрик в InfluxDB

**Шаг 1:** Установите клиент Influx DB

`pip install -r requirements.txt`

или

`pip install influxdb-client[ciso]`

**Шаг 2:** Настройте url, org, token и bucket в config.py.

Больше информации - https://github.com/influxdata/influxdb-client-python

**Шаг 3:** Вызовите log_metrics

```python
log_metrics(timestamp, collection, metrics, tags)
```

**Пример использования:**

```python
from src import log_metrics
from datetime import datetime

log_metrics(datetime.utcnow(), "test", {
        "balance": 1000,
        "pnl": -500,
        "margin": 500
    },
    {
        "account": "binance1",
        "pair": "BTCUSDT",
        "base_asset": "BTC",
        "quote_asset": "USDT",
        "strategy": "SMA"
    })
```

По умолчанию логируются метрики, связанные с маржой (коллекция = margin) из WebSocket-потока Binance и метрики для ордеров с использованием limit chaser (коллекция = chaser).

**Сопоставление log_metrics с InfluxDB:**

1. collection -> measurement
2. metrics -> fields
3. tags -> tags
